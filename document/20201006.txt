scope (객체) : 데이터를 저장할 수있는 공간
** 사용 이유
요청을 접수받는 모듈(servlet - controller)과
응답을 생성하는 모듈(jsp - view) 이 나뉘어짐

요청을 받는 쪽에서 로직을 처리하고 처리결과를 view 그려낼려면 데이터를 넘겨줘야함
그때 사용 하는 것이 scope 객체

servlet 에서 request, session setAttribute를 통해 저장하고
jsp에서는 request, session getAttrbiute를 통해 데이터를 확인, 사용

scope 객체 종류
page < request < session < application

request : 요청이 처리되고 응답이 생성될 때까지 설정한 속성이 유효
session : 사용자가 유지되는 시간동안 서버상에 설정이 유효
          사용자 마다 별도의 저장공간을 할당 받음
          불필요하게 session 속성을 저장하는 것은 서버 입장에서는 부하
          모든 페이지에서 자주 사용하는 데이터들만 한정하여 저장
          (ex : 로그인시 사용자 정보)
application : 서버상에 하나만 존재하는 공간
              모든 사용자가 동일한 데이터를 보게 된다
